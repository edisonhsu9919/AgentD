📋 项目概述

智能职业分类系统是一个基于大语言模型(LLM)的多功能职业分析平台，采用模块化微服务架构设计。系统支持职业风险评估、行业分类、技能分析等多种分类任务，提供RESTful API和命令行界面，兼容多种LLM提供商（本地vLLM、OpenAI、通义千问等）。

📊 支持的分类任务

🛡️ 职业风险评估
risk_assessment

基于作业环境和工作内容的 6 级风险分类体系

  • 1类 - 极低风险：办公室岗位
  • 2类 - 低风险：辅助性岗位
  • 3类 - 中等风险：农业自然作业
  • 4类 - 中高风险：工厂基础危险
  • 5类 - 高风险：有毒有害化工
  • 6类 - 极高风险：矿井爆破高空
🏭 行业分类
industry_classification

基于国民经济行业分类标准的 20 类行业归类（GB/T 4754-2017）

  • A类 - 农、林、牧、渔业
  • B类 - 采矿业
  • C类 - 制造业
  • D类 - 电力、热力、燃气及水生产
  • E类 - 建筑业
  • ... 共 20 个行业类别
🎓 技能分析
skill_analysis

基于工作复杂度和技能要求的 5 级技能分析

  • 初级 - 基础技能要求，简单重复性工作
  • 中级 - 专业技能要求，独立完成任务
  • 高级 - 高级专业技能，复杂问题解决
  • 专家级 - 领域专家技能，创新与研发
  • 管理级 - 管理和领导技能，战略规划
🎨 自定义任务
custom

支持用户自定义分类任务和模板

  • 自定义分类等级和描述
  • 灵活的提示词模板
  • 通过 API 动态创建
  • 支持复杂业务规则
  • 可保存和复用模板

🏗️ 项目架构 v2.0

🌐 API 层 (src/api/)

  • app.py: FastAPI 应用主文件，路由配置
  • models.py: Pydantic 数据模型定义
  • routes/: 分类、文件、配置等 API 路由
  • 提供完整的 RESTful API 和自动文档（Swagger UI）

🎯 核心逻辑层 (src/core/)

  • classifier.py: UniversalClassifier 通用分类引擎
  • batch_processor.py: 批处理器，支持异步/并发
  • 支持多种分类任务的统一处理接口
  • 错误处理和重试机制

🤖 LLM 集成层 (src/llm/)

  • client.py: LLMClient 通用客户端接口
  • providers/: 各 LLM 提供商实现
  • 支持本地 vLLM、OpenAI、Qwen、自定义 API
  • 运行时动态切换提供商

📝 提示词管理 (src/prompts/)

  • manager.py: PromptManager 模板管理器
  • templates/: 各分类任务的提示词模板
  • 支持动态创建和管理自定义模板
  • 统一的提示词接口和响应解析

⚙️ 配置管理 (config/)

  • settings.py: 集中式配置管理
  • 支持环境变量 (.env) 配置
  • 运行时动态配置更新
  • 多环境配置支持

🚀 应用入口

  • main.py: CLI 命令行工具入口
  • 支持 server、single、batch、tasks、test 等命令
  • 统一的命令行参数解析

🚀 快速开始

1. 环境准备

# 克隆项目 git clone <repository_url> cd employee_risk_classify # 安装依赖 pip install -r requirements.txt # 配置环境变量 cp .env.example .env # 编辑 .env 文件配置你的 LLM 提供商信息

2. 启动 API 服务

# 启动服务器 python main.py server # 访问 API 文档 # http://localhost:8000/docs

3. 命令行使用

# 单条分类 python main.py single "软件工程师" -c "科技公司" -t "risk_assessment" # 批量处理文件 python main.py batch input.xlsx output.xlsx -t "risk_assessment" # 查看可用分类任务 python main.py tasks # 测试 LLM 连接 python main.py test

4. API 调用示例

# 单条分类 API curl -X POST "http://localhost:8000/api/v1/classification/single" \ -H "Content-Type: application/json" \ -d '{ "job_title": "软件工程师", "company_name": "科技公司", "task_type": "risk_assessment" }' # 批量分类 API curl -X POST "http://localhost:8000/api/v1/classification/batch" \ -H "Content-Type: application/json" \ -d '{ "items": [...], "task_type": "risk_assessment" }'

🔧 API 接口详情

分类接口

接口 方法 说明
/api/v1/classification/single POST 单条职业数据分类
/api/v1/classification/batch POST 批量职业数据分类
/api/v1/file/upload POST 上传 Excel/CSV 文件处理
/api/v1/file/status/{task_id} GET 查询处理状态和进度
/api/v1/file/download/{task_id} GET 下载处理结果文件

配置管理接口

接口 方法 说明
/api/v1/llm/config GET 获取当前 LLM 配置
/api/v1/llm/config POST 更新 LLM 配置（运行时）
/api/v1/templates GET 列出所有可用分类模板
/api/v1/templates/custom POST 创建自定义分类模板
/api/v1/templates/custom/{name} DELETE 删除自定义模板

⚙️ 配置说明

环境变量配置

变量名 默认值 说明
CLASSIFIER_LLM_PROVIDER local_vllm LLM 提供商：local_vllm | openai | qwen | custom
CLASSIFIER_LLM_BASE_URL http://localhost:8000/v1 LLM API 基础地址
CLASSIFIER_LLM_MODEL model_name 使用的模型名称
CLASSIFIER_LLM_API_KEY - API 密钥（外部服务需要）
CLASSIFIER_HOST 0.0.0.0 API 服务监听地址
CLASSIFIER_PORT 8000 API 服务端口
CLASSIFIER_UPLOAD_PATH uploads 文件上传目录
CLASSIFIER_OUTPUT_PATH outputs 处理结果输出目录

LLM 提供商配置示例

本地 vLLM (推荐)

CLASSIFIER_LLM_PROVIDER=local_vllm CLASSIFIER_LLM_BASE_URL=http://localhost:8000/v1 CLASSIFIER_LLM_MODEL=/path/to/models/Qwen3-8b-AWQ

OpenAI

CLASSIFIER_LLM_PROVIDER=openai OPENAI_API_KEY=your_openai_api_key CLASSIFIER_LLM_MODEL=gpt-3.5-turbo

通义千问

CLASSIFIER_LLM_PROVIDER=qwen QWEN_API_KEY=your_qwen_api_key CLASSIFIER_LLM_MODEL=qwen-turbo

🛠️ 开发指南

添加自定义分类任务

  1. 在 src/prompts/templates/ 创建新的提示词模板类
  2. 继承 BasePromptTemplate 基类
  3. 实现 get_prompt() 和 parse_response() 方法
  4. 在 PromptManager 中注册新模板
  5. 通过 API 或 CLI 使用新任务

扩展 LLM 提供商

  1. 在 src/llm/providers/ 创建新的提供商类
  2. 继承 BaseLLMProvider 基类
  3. 实现 generate() 方法
  4. 在 PROVIDER_MAP 中注册
  5. 通过环境变量切换到新提供商

性能优化建议

  • 并发处理: 批量任务调整 max_concurrent 参数
  • 异步架构: FastAPI 异步处理提升响应速度
  • 缓存机制: 支持结果缓存减少重复计算
  • 错误恢复: 单项失败不影响批量处理
  • 资源管理: 合理配置 LLM 服务的资源限制

🔍 故障排除

常见问题

  • LLM 连接失败: 检查 API 配置和网络连接，确认服务运行状态
  • 文件处理超时: 调整并发数或分批处理，检查 LLM 性能
  • 内存不足: 适当减少批处理大小，优化模型配置
  • 分类结果不准确: 调整提示词模板，使用更强大的模型
  • API 响应慢: 检查 LLM 服务性能，考虑使用更快的模型
